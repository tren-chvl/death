    Chercher le segment exécutable  
    Il parcourt les Program Headers et repère le segment PT_LOAD qui a les permissions R + X (celui qui contient le code).

    Vérifier s’il existe un espace libre  
    Il regarde s’il y a un “trou” entre la fin de ce segment et le début du suivant.

    Placer des données dans cet espace libre  
    Si l’espace est suffisant, il y met un bloc de données (son “Death”), comme un append dans une zone vide du fichier.

    Étendre les tailles du segment  
    Il augmente p_filesz et p_memsz pour que le loader charge aussi cette zone supplémentaire.

    Changer l’entry point  
    Il modifie e_entry pour que l’exécution commence dans la zone qu’il a ajoutée.

    Exécuter un petit stub puis revenir au programme  
    Son code fait quelque chose (souvent rien de spécial), puis saute vers l’ancien entry point pour lancer le programme original.

    Plan B : utiliser un PT_NOTE  
    Si aucun espace libre n’existe, il prend un segment PT_NOTE (non chargé) et le transforme conceptuellement en segment chargé/exécutable pour y mettre son bloc.
